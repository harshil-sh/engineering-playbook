# Python Best Practices

Python is most effective when code stays explicit, testable, and easy to reason about.

## Principles
- Prefer readability over cleverness.
- Keep modules focused and functions small.
- Use type hints for clarity and tooling support.
- Isolate side effects at the edges.
- Write tests for behaviour, not implementation details.

## Production checklist
- Format and lint consistently.
- Use virtual environments and pinned dependencies when appropriate.
- Structure packages so imports remain predictable.
- Log meaningful operational context.
- Treat scripts as software when they become business critical.
