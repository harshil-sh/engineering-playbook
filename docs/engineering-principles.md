# Engineering Principles

These are the principles I return to when making technical and delivery decisions.

## Principles I optimise for
1. **Clarity first** — systems should be understandable.
2. **Reliability by design** — production behaviour matters more than local success.
3. **Simple over clever** — complexity must justify itself.
4. **Ownership end to end** — build, release, observe, and improve.
5. **Document key decisions** — future teams deserve context.
6. **Continuous improvement** — every release should teach something.

## In practice
These principles influence architecture, code review, delivery planning, and team collaboration.
