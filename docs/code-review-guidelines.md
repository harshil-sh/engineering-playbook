# Code Review Guidelines

Code review is a quality practice and a collaboration practice.

## Review for
- Correctness
- Maintainability
- Security and privacy concerns
- Performance impact
- Test coverage and release safety
- Naming, clarity, and documentation

## Reviewer mindset
Aim to improve the code and help the author. Be direct, respectful, and specific about risks or trade-offs.

## Author checklist
- Keep pull requests focused.
- Explain context and impact.
- Add tests or justify why they are not needed.
- Highlight risky areas and rollback considerations.
